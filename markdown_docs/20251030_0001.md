## REST 

### 1. REST 제약조건

```aiignore
1) 인터페이스 일관성
외부에서 호출가능한 인터페이스가 제공되어야 한다.

2) 무상태(Stateless)
호출한 클라이언트의 상태를 서버에서 저장하지 않는다.

3) 캐시 처리 가능(Cacheeable)
보관된 데이터를 제공하는 방식으로 동작할 수 있다.

4) 계충화(Layed System)
데이터를 제공하는 서비스의 구조적인 제한이 없다.
인증이나 암호화 등을 자유롭게 추가할 수 있다.


[요약]
"REST 방식은 상태를 유지하지 않고, 호출 가능한 경로를 설계해야 한다"
```

### 2.REST 가이드
```aiignore
1) 자원의 식별 
원하는 데이터를 찾을 수 있는 고유의 주소가 존재해야 한다.

2) 메시지를 통한 리소스의 조작
클라이언트가 보내는 정보 안에는 자원 처리에 대한 정보가 포함되어야 한다.

3) 자기 서술적 메세지
요청/전송하는 정보에 대한 추가적인 데이터가 포함되어야 한다.
예를 들어 해당 데이터가 어떤 포맷으로 구성되어 있는지, 길이는 어떻게 되는지 등과 같이 메세지를 설명하는 정보가 함께 포함되어야 한다.

4) 하이퍼미디어 제약
외부에서 링크 등을 통해서 애플리케이션 내 자원들의 상태를 변경할 수 있어야 한다.


[예제]
여행사 여행상품을 REST 에 적용 


1) 자원의 식별 
각 여행 상품은 고유의 주소가 있어야 한다.

2) 메시지를 통한 리소스의 조작
여행상품에 대한 CRUD 에 대해서 GET 방식은 조회할 때 사용한다.

3) 자기 서술적 메세지
상품 데이터는 HTTP 응답 헤더에 여러 내용을 포함한다.
데이터 뿐만 아니라 데이터의 종류나 크기 등과 같은 추가적인 데이터를 전송 

4) 하이퍼미디어 제약
<form> 태그 만으로도 해당 데이터 수정이나 삭제가 가능하다.
```


### 3. REST 방식 API 설계

```aiignore
예 : 
/api/v1/todos
API 설계에는 v1, v2 와 같이 해당 API의 버전을 명시해 주는 것이 나중에 변경시 유용하다.


1) 단순 CRUD
/api/vi/todos/123   GET     -- 123 식별데이터로 찾을 수 있는 todo 정보를 반환 
/api/vi/todos/123   PUT     -- 123 todo 정보 수정
/api/vi/todos/123   DELETE  -- 123 todo 삭제
/api/vi/todos       POST    -- 새로운 todo 정보 추가


2) 쿼리스트링
REST 방식에서 쿼리스트링은 자원의 고유 식별자와 조금 다르게 사용되는데,
쿼리스트링은 동일한 문자열 이라고 해도 다른 결과가 발생하는 경우에 사용한다.

예를 들어 페이지번호의 경우는 오늘 1페이지의 내용이 시간이 지나면 변경될 수 있기 때문에, 
고유한 식별값을 이용하지 않고 쿼리스트링을 통해서 처리한다.

결론적으로 매번 동일한 데이터를 나타내지 않는다면 쿼리스트링으로 처리한다.

/api/v1/todos?page=1                        GET -- 1페이지 정보
/api/v1/todos?page=1&size=10                GET -- 10개씩 보여주는 1페이지
/api/v1/todos?page=1&keyword=AAA&type=N     GET -- 제목(N) 으로 검색하고 keyword 가 AAA 가 있는 목록 1페이지 


3) 계층적인 데이터
가령 todo 의 리뷰나, 댓글 데이터를 조회해야 핟나면, 기존의 /todos/123 뒤에 추가경로를 이용해서 구성할 수 있다.

/api/v1/todos/123/reviews?page=1    GET -- 123번 todo 의 리뷰/댓글 중 1페이지(page) 의 데이터
/api/v1/todos/123/reviews/33        GET -- 123번 todo 의 리뷰/댓글 중 33번 리뷰 데이터 조회
```

```aiignore
Spring Web 은 Spring Boot 애플리케이션을 웹 서버로 동작시키기 위한 의존성입니다.
```

| **jackson-databind**               | JSON 직렬화/역직렬화 (Controller에서 JSON 응답할 때 사용) |
| **spring-boot-starter-validation** | Bean Validation (입력값 검증용)                  |
