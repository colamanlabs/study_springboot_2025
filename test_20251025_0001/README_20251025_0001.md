## @SpringBootApplication

SpringBoot 어플리케이션에는 SpringBootApplication 어노테이션이 붙는다.


```aiignore

@Retention(RetentionPolicy.RUNTIME)
@Documented
@Inherited
@SpringBootConfiguration
@EnableAutoConfiguration
@ComponentScan(
    excludeFilters = {@Filter(

    classes = {TypeExcludeFilter.class}

    type = FilterType.CUSTOM,

)}
)
public @interface SpringBootApplication {
...
}
```
@Target({ElementType.TYPE})
- Type 으로서 클래스 선언 위에 붙일 수 있고,

@Retention(RetentionPolicy.RUNTIME)
- 적용범위가 런타임시에만 해당되며

@Documented
- javaodc 문서화 될 수 있고

@Inherited
- 상속 가능하며

@SpringBootConfiguration
- @Configuration 기능에 SpringBoot 의 고유기능울 추가함,
- @Configuration 포함하므로, 선언한 클래스에서 @Bean 등록 가능함
- 설정파일 application.properties 처리도 해주지만, 설정파일 application.properties 은 컴포넌트 스캔범위와 다른 독립범위를 사용하며, 작업디렉토리, 클래스패스의 root 경로를 사용함

@EnableAutoConfiguration
- SpringBoot 자동 설정 기능 
- 자동으로 클래스들을 스캔해서, 필요한 Bean 을 자동 등록한다는 의미
- 가령 spring-boot-starter-web 를 추가하면, DispatcherServlet, Tomcat, Jackson 등이 자동으로 등록

@ComponentScan
- 컴포넌트 스캔
